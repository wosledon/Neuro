# 目录与结构规范

目的：提供一个清晰的一致性的目录结构，方便后续扩展与维护。

建议的目录结构：

- Abstractions/
  - 放置接口、契约（如 `IVectorStore`、`VectorRecord` 等）与公共模型。
- Stores/
  - 存放具体的本地实现（例如 `LocalVectorStore`、`LockFreeVectorStore`）。
- Providers/
  - 提供者注册与适配器实现（第三方/远程 DB 的适配器工厂）。
- Extensions/
  - DI/Host 扩展方法，例如 `AddVectorStore` 与 `AddVectorStoreProvider`。
- docs/
  - 存放此类文档、设计说明与迁移指南。

命名与文档规范：
- 代码注释（XML 文档注释）使用中文，简洁明了，说明行为、输入输出与异常情况。
- 异常消息也使用中文，便于本地化日志与排错。

迁移建议：
- 由于 C# 文件的命名空间与物理位置无强耦合，建议逐步移动文件到上述目录：
  1. 在目标目录下创建文件并复制实现。
  2. 运行单元测试，确保编译与测试通过后再删除源文件。

其他建议：
- 保持 public API 稳定，在修改异常或文案时尽量不改变方法签名或行为。
- 为每个提供者添加独立的 README（说明支持的配置与限制）。

如需我代为执行实际的文件迁移（移动/重命名），我可以按步骤进行，但会先创建迁移分支并运行测试以确保安全。
